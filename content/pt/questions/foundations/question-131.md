question: "Qual afirmação descreve melhor a diferença entre clonar um repositório e criar um fork de um repositório?"  
draft: false  
---

> **Mais informações**: [Sobre forks](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/about-forks)

1. [ ] Quando você clona um repositório, as alterações que você faz push vão diretamente para o repositório original. Criar um fork exige uma cópia local separada.  
  > Clonar e criar um fork resultam em cópias locais separadas, mas fazer push para um clone exige a permissão correta no repositório original.  
1. [x] Criar um fork cria uma cópia no seu GitHub, enquanto clonar cria uma cópia local no seu computador sem vincular a propriedade à sua conta.  
  >
