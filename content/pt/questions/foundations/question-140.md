---
title: "Pergunta 140"
question: "Sua equipe está se preparando para implantar uma nova funcionalidade usando branches do GitHub. O que deve ser feito para garantir que o branch de funcionalidade esteja pronto para implantação?"
---

> **Mais informações**: [Sobre pull requests](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-pull-requests)
1. [ ] Fazer commit de todas as alterações diretamente no branch main sem revisão.
1. [ ] Excluir o branch de funcionalidade para finalizar o processo de implantação.
1. [x] Garantir que todas as alterações sejam revisadas e aprovadas por meio de um pull request antes de fazer merge no branch main.
