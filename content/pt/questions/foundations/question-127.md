---
title: "Questão 127"
question: "Qual declaração descreve melhor uma grande diferença entre os novos GitHub Projects e o GitHub Projects Classic?"
---

> **Mais informações**: [Sobre projetos](https://docs.github.com/en/issues/planning-and-tracking-with-projects/learning-about-projects/about-projects#differences-from-projects-classic)

1. [x] Os novos GitHub Projects fornecem uma interface flexível semelhante a uma planilha, campos personalizados e sincronização bidirecional de problemas que não estavam disponíveis no Projects Classic.  
1. [ ] GitHub Projects Classic tem uma IA integrada que fecha automaticamente pull requests, enquanto os novos GitHub Projects não possuem isso.  
1. [ ] GitHub Projects Classic suporta varredura de código e verificação de segredo, enquanto os novos GitHub Projects não.  
1. [ ] Os novos GitHub Projects permitem apenas gerenciar problemas abertos, sem suporte a pull requests.
