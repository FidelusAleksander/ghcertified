---


> https://docs.github.com/en/get-started/quickstart/github-flow
1. [x] Refere-se a um fluxo de trabalho baseado em branch onde os desenvolvedores criam uma branch para cada nova alteração em que estão trabalhando e, em seguida, abrem pull requests para que seu código seja revisado, testado e mesclado na branch main.
1. [ ] É uma prática onde, toda vez que um commit é efetuado na branch main, uma nova versão é automaticamente criada e implantada.
1. [ ] É uma prática onde, toda vez que um commit é efetuado na branch main, um conjunto de pipelines de CI/CD é acionado para executar testes e validar as alterações de código.
