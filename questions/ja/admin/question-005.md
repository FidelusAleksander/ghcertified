---
question: "次のうち、GitHub Enterprise Serverの診断ファイルを生成し共有する正しい方法はどれですか？"
documentation: "https://docs.github.com/en/enterprise-server/support/contacting-github-support/providing-data-to-github-support#creating-and-sharing-diagnostic-files"
---

- [x] インスタンスの診断情報を取得するために、`ghe-diagnostics` コマンドラインユーティリティを使用する。
- [x] 管理コンソールのサポートタブに移動し、`Download diagnostics info` をクリックして診断ファイルを作成する。
- [ ] 診断ファイルを作成してアップロードするために、`ghe-support-bundle` コマンドを使用する。
- [ ] GitHubモバイルアプリを使用して診断ファイルを生成する。
- [ ] GitHub Supportに診断ファイルを自動的にアップロードするために、`ghe-diagnostics -u` コマンドを実行する。
