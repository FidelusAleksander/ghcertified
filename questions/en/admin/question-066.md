---
question: "Why might webhooks be considered an efficient alternative to the audit log or API polling in certain use cases within GitHub organizations?"
documentation: "https://docs.github.com/en/organizations/keeping-your-organization-secure/managing-security-settings-for-your-organization/reviewing-the-audit-log-for-your-organization"
---

- [x] Webhooks provide real-time notifications to your server when specific events occur, which can be more efficient than searching the audit log or polling the API for those events.
- [ ] Webhooks allow organization admins to manually review and approve each event before it is logged in the organization's audit trail for compliance.
- [ ] They provide a more detailed and comprehensive log of events than the audit log, including full request payloads and response data.
- [ ] Webhooks are the only method to receive notifications about private repository actions, as the audit log does not track private repos.
