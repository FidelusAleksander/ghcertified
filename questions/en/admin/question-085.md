- [x] Choose self-hosted runners for customized environments or GitHub-hosted for automatic updates and clean instances.
- [ ] Always prefer GitHub-hosted runners for all workloads to ensure maximum security.
- [ ] Use only self-hosted runners, regardless of workload complexity, to minimize costs.
- [ ] Restrict to using GitHub-hosted runners with custom hardware configurations for better performance.
