---
question: "How can GitHub Apps react to specific events, and what are some examples of these events?"
documentation: "https://docs.github.com/en/apps/creating-github-apps/registering-a-github-app/using-webhooks-with-github-apps#subscribing-to-webhook-events"
---

- [x] GitHub Apps subscribe to events through webhooks, which notify the app of specific actions like pull request openings or issue creations.
- [ ] GitHub Apps use polling to periodically check for new events on GitHub.
- [ ] GitHub Apps receive all events by default and must filter them in the app's code.
- [ ] GitHub Apps use a WebSocket connection to receive real-time notifications of all repository events.
