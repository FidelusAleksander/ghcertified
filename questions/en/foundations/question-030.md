question: "If you stop your GitHub Codespace environment can you come back to the changes later if you haven't committed them?"
documentation: "https://docs.github.com/en/codespaces/getting-started/understanding-the-codespace-lifecycle#stopping-a-codespace"

- [x] Yes, that's the default behavior when stopping and then starting a GitHub Codespace
- [ ] No, these changes will be lost. You need to commit them before stopping the Codespace.
