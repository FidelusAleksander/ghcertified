---
question: "¿Qué afirmación describe mejor una diferencia importante entre los nuevos GitHub Projects y GitHub Projects Classic?"
documentation: "https://docs.github.com/en/issues/planning-and-tracking-with-projects/learning-about-projects/about-projects#differences-from-projects-classic"
---

- [x] Los nuevos GitHub Projects ofrecen una interfaz flexible similar a una hoja de cálculo, campos personalizados y sincronización bidireccional de issues que no estaban disponibles en Projects Classic.  
- [ ] GitHub Projects Classic tiene una IA integrada que cierra automáticamente los pull requests, mientras que los nuevos GitHub Projects no lo hacen.  
- [ ] GitHub Projects Classic admite el análisis de código y el escaneo de secretos, mientras que los nuevos GitHub Projects no lo hacen.  
- [ ] Los nuevos GitHub Projects solo permiten gestionar issues abiertas sin admitir pull requests.
