---
question: "Los flujos de trabajo de GitHub Actions se activan por eventos. ¿Cuáles de estos son eventos válidos que GitHub Actions soporta?"
documentation: "https://docs.github.com/en/actions/using-workflows/events-that-trigger-workflows#about-events-that-trigger-workflows"
---

- [x] Se realiza un commit en una branch
- [x] Se abre un pull request
- [ ] Se realiza un cambio en la configuración del repository
- [ ] Se detecta una nueva vulnerabilidad en una dependencia
> Este no es un evento válido que podría activar un flujo de trabajo. Es una característica de Dependabot.
- [ ] Se agrega un nuevo miembro al repository
