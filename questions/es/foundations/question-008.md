---
question: "¿Cuáles de estos flujos de trabajo son automatizaciones integradas en GitHub Projects?"
documentation: "https://docs.github.com/en/issues/planning-and-tracking-with-projects/automating-your-project/using-the-built-in-automations"
---

- [x] Cuando los issues o pull requests en tu proyecto son cerrados, su estado se establece como Done.
- [x] Cuando los pull requests en tu proyecto son fusionados, su estado se establece como Done.
- [ ] Cuando los issues o pull requests son abiertos en tu proyecto, su estado se establece como Done.
- [ ] Cuando se agrega un nuevo colaborador a un repository, los issues no asignados se le asignan.
- [ ] Cuando se activa una GitHub Action, se crea un nuevo elemento en tu GitHub Project.
