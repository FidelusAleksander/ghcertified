---
question: "Varios repositorios públicos a los que estás contribuyendo no tienen habilitada la opción de protección contra el escaneo de secretos al realizar un push. ¿Qué puedes hacer para evitar accidentalmente subir secretos a estos repositorios?"
documentation: "https://docs.github.com/en/code-security/secret-scanning/push-protection-for-users#about-push-protection-for-users"
---

- [x] Habilitar `Push protection for yourself` en la configuración de tu cuenta personal de GitHub
- [ ] Descargar el complemento web de protección de push de GitHub
- [ ] No es posible, la protección de push debe habilitarse a nivel de repositorio, organización o enterprise
- [ ] Agregar los archivos que contienen secretos al archivo `.gitignore` en todos los repositorios
