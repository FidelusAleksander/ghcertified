---
question: "¿Cuáles son las principales implicaciones de habilitar el inicio de sesión único SAML (SSO) para una organización en GitHub Enterprise Cloud?"
documentation: "https://docs.github.com/en/enterprise-cloud@latest/authentication/authenticating-with-single-sign-on/about-authentication-with-single-sign-on"
---

- [x] El SSO SAML permite a los propietarios de la organización controlar y asegurar el acceso a los recursos de la organización.
- [ ] Habilitar el SSO SAML aprovisiona automáticamente nuevas cuentas de GitHub para todos los usuarios del IdP.
- [ ] Una vez habilitado el SSO SAML, los repositorios públicos se vuelven inaccesibles.
- [ ] Habilitar el SSO SAML requiere que los miembros se autentiquen a través de GitHub, no del IdP.
