---
question: "¿Cuál es el propósito de SCIM y la sincronización de equipos en GitHub?"
documentation: "https://docs.github.com/en/enterprise-cloud@latest/organizations/managing-saml-single-sign-on-for-your-organization/managing-team-synchronization-for-your-organization"
---

- [x] La sincronización de equipos conecta los equipos de GitHub con los grupos del IdP para la gestión de membresías, dependiendo del inicio de sesión único de SAML, mientras que SCIM automatiza la provisión y gestión de identidades entre sistemas, incluyendo la creación y actualización de identidades de usuario y accesos.
- [ ] La sincronización de equipos es un proceso automatizado que permite a los usuarios gestionar sus perfiles públicos y repositorios mediante una interfaz específica de GitHub.
- [ ] La sincronización de equipos se utiliza para la seguridad interna de datos y la gestión de información de usuarios, mientras que SCIM es una interfaz para la personalización de perfiles de usuario y repositorios.
- [ ] Tanto la sincronización de equipos como SCIM en GitHub se utilizan para servicios de provisión de usuarios, invitando a no miembros a unirse a organizaciones y gestionando perfiles públicos y repositorios.
