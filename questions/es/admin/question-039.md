---
question: "¿Cómo puede un administrador de la organización establecer permisos predeterminados para nuevos miembros en una organización de GitHub?"
documentation: "https://docs.github.com/en/organizations/managing-user-access-to-your-organizations-repositories/managing-repository-roles/setting-base-permissions-for-an-organization#setting-base-permissions"
---

- [x] En la sección `Access` de los ajustes de la organización, selecciona `Member privileges`, luego, bajo `Base permissions`, elige un nivel de permisos y confirma el cambio.
- [ ] Los permisos predeterminados son determinados automáticamente por GitHub en función de la actividad de la organización y no pueden configurarse manualmente.
- [ ] Configurando permisos individualmente para cada nuevo miembro a medida que se les agrega a la organización, sin que haya un nivel predeterminado disponible.
- [ ] Los permisos predeterminados para nuevos miembros se configuran mediante una llamada a la API de GitHub, especificando el nivel de permiso base deseado para la organización.
