---
question: "¿Cuál de estas prácticas puede ayudar a evitar el compromiso de datos o archivos sensibles en un repositorio de git?"
documentation: "https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/removing-sensitive-data-from-a-repository#avoiding-accidental-commits-in-the-future"
---

- [x] Agregar archivos o patrones sensibles a un archivo `.gitignore`
- [ ] Cambiar tus contraseñas regularmente
- [ ] Desactivar el acceso de push al repositorio
- [ ] Usar `git add .` antes de todos los commits
- [ ] Desactivar todas las reglas de `.gitignore` para asegurar que cada archivo sea rastreado
