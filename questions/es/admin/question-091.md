---
question: "¿Cómo puedes usar bóvedas de terceros para gestionar secretos en GitHub Actions?"
documentation: "https://developer.hashicorp.com/vault/docs/platform/github-actions"
---

> https://bitwarden.com/help/github-actions-integration/
> https://docs.keeper.io/keeperpam/secrets-manager/integrations/github-actions
- [x] Almacena los secretos en la bóveda de terceros y autentícate desde tu workflow (a través de la acción del proveedor/CLI) usando credenciales de acceso almacenadas como secretos de GitHub Actions para recuperarlos en tiempo de ejecución.
- [ ] La integración directa con bóvedas de terceros es compatible de forma nativa dentro de GitHub Actions sin necesidad de configuración adicional.
- [ ] Las bóvedas de terceros deben ser accedidas incrustando sus claves API dentro de los archivos del workflow y utilizando la API o CLI de ese proveedor.
- [ ] GitHub Actions no admite el uso de bóvedas de terceros para la gestión de secretos; todos los secretos deben almacenarse dentro de GitHub.
