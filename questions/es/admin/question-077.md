---
question: "¿Cómo pueden las aplicaciones de GitHub reaccionar a eventos específicos y cuáles son algunos ejemplos de estos eventos?"
documentation: "https://docs.github.com/en/apps/creating-github-apps/registering-a-github-app/using-webhooks-with-github-apps#subscribing-to-webhook-events"
---

- [x] Las aplicaciones de GitHub se suscriben a eventos a través de webhooks, los cuales notifican a la aplicación sobre acciones específicas como la apertura de pull requests o la creación de issues.
- [ ] Las aplicaciones de GitHub utilizan polling para verificar periódicamente nuevos eventos en GitHub.
- [ ] Las aplicaciones de GitHub reciben todos los eventos por defecto y deben filtrarlos en el código de la aplicación.
- [ ] Las aplicaciones de GitHub utilizan una conexión WebSocket para recibir notificaciones en tiempo real de todos los eventos del repositorio.
