---
question: "¿Cuáles son las características y funcionalidades clave de los equipos dentro de una organización de GitHub?"
documentation: "https://docs.github.com/en/organizations/organizing-members-into-teams/about-teams"
---

- [x] Los equipos en las organizaciones de GitHub permiten gestionar el acceso y las notificaciones para grupos que reflejan la estructura de la empresa, con capacidades para acceso de administrador, lectura o escritura a repositorios, menciones para notificaciones y el uso de equipos anidados para gestión jerárquica.
- [ ] Los equipos son únicamente para redes sociales dentro de GitHub, permitiendo que los miembros de una organización se comuniquen y compartan actualizaciones sin ningún impacto en el acceso a repositorios o las notificaciones organizacionales.
- [ ] Los equipos de GitHub son idénticos a los miembros de la organización, sin características o funcionalidades distintivas, como gestión de acceso o notificaciones.
- [ ] Los equipos dentro de GitHub pueden incluir colaboradores externos y se utilizan principalmente para comunicación externa, sin ninguna estructura interna organizativa o permisos de acceso a repositorios.
