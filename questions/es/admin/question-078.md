- [x] Usando webhooks para suscribirse a información sobre los trabajos y ejecuciones de flujos de trabajo, y potencialmente utilizando un sistema de archivo de datos.
- [x] Usando métricas de uso y rendimiento de GitHub Actions disponibles a nivel de organización y repositorio.
> Las métricas de GitHub Actions son accesibles desde la pestaña Insights a nivel de organización y repositorio, proporcionando datos de uso y rendimiento como tiempos de ejecución, tiempos en cola y tasas de fallos.
- [ ] Contando manualmente las ejecuciones y trabajos de flujos de trabajo en cada repositorio al final de cada mes.
- [ ] El uso de GitHub Actions no se puede rastrear; las empresas deben estimarlo basándose en la retroalimentación de los desarrolladores.
