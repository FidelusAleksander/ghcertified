---
question: "Które stwierdzenie najlepiej opisuje główną różnicę między nowymi GitHub Projects a GitHub Projects Classic?"
documentation: "https://docs.github.com/en/issues/planning-and-tracking-with-projects/learning-about-projects/about-projects#differences-from-projects-classic"
---

- [x] Nowe GitHub Projects oferują elastyczny interfejs przypominający arkusz kalkulacyjny, pola niestandardowe i dwukierunkową synchronizację problemów, co nie było dostępne w Projects Classic.  
- [ ] GitHub Projects Classic mają wbudowaną sztuczną inteligencję, która automatycznie zamyka pull requests, podczas gdy nowe GitHub Projects tego nie robią.  
- [ ] GitHub Projects Classic obsługują skanowanie kodu i skanowanie sekretów, podczas gdy nowe GitHub Projects tego nie robią.  
- [ ] Nowe GitHub Projects pozwalają zarządzać tylko otwartymi problemami bez obsługi pull requests.
