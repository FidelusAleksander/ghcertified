---
question: "Jak działa synchronizacja między projektami GitHub, zgłoszeniami i pull requestami?"
documentation: "https://docs.github.com/en/issues/planning-and-tracking-with-projects/learning-about-projects/about-projects#staying-up-to-date"
---

- [x] Aktualizacje zgłoszeń i pull requestów są automatycznie odzwierciedlane w GitHub Projects. Integracja działa w obie strony, co oznacza, że gdy zmieniasz informacje o pull requestach lub zgłoszeniach w swoim projekcie, dane te są aktualizowane również w pull requestach lub zgłoszeniach.
- [ ] Aktualizacje zgłoszeń i pull requestów nie są automatycznie odzwierciedlane w GitHub Projects; muszą być ręcznie inicjowane przez użytkownika.
- [ ] Synchronizacja informacji działa tylko w jednym kierunku - od projektu do zgłoszeń i pull requestów. Aktualizacje zgłoszeń i pull requestów nie są automatycznie odzwierciedlane w elementach projektu GitHub.
- [ ] Synchronizacja informacji działa tylko w jednym kierunku - od zgłoszeń i pull requestów do projektów. Aktualizacje elementów projektu nie są automatycznie odzwierciedlane w zgłoszeniach i pull requestach.
