---
question: "Jak aplikacje GitHub mogą reagować na konkretne zdarzenia i jakie są przykłady tych zdarzeń?"
documentation: "https://docs.github.com/en/apps/creating-github-apps/registering-a-github-app/using-webhooks-with-github-apps#subscribing-to-webhook-events"
---

- [x] Aplikacje GitHub subskrybują zdarzenia za pośrednictwem webhooków, które powiadamiają aplikację o konkretnych działaniach, takich jak otwieranie pull requestów czy tworzenie issue.
- [ ] Aplikacje GitHub używają odpytywania (polling), aby okresowo sprawdzać nowe zdarzenia na GitHub.
- [ ] Aplikacje GitHub otrzymują wszystkie zdarzenia domyślnie i muszą je filtrować w kodzie aplikacji.
- [ ] Aplikacje GitHub używają połączenia WebSocket do otrzymywania powiadomień w czasie rzeczywistym o wszystkich zdarzeniach w repository.
