---
question: "Dlaczego webhooki mogą być uważane za wydajną alternatywę dla dziennika inspekcji lub odpytywania API w określonych przypadkach użycia w organizacjach GitHub?"
documentation: "https://docs.github.com/en/organizations/keeping-your-organization-secure/managing-security-settings-for-your-organization/reviewing-the-audit-log-for-your-organization"
---

- [x] Webhooki zapewniają powiadomienia w czasie rzeczywistym na Twój serwer, gdy wystąpią określone zdarzenia, co może być bardziej wydajne niż przeszukiwanie dziennika inspekcji lub odpytywanie API dla tych zdarzeń.
- [ ] Webhooki pozwalają administratorom organizacji ręcznie przeglądać i zatwierdzać każde zdarzenie przed jego zarejestrowaniem.
- [ ] Zapewniają bardziej szczegółowy i kompleksowy rejestr zdarzeń niż dziennik inspekcji.
- [ ] Webhooki są jedyną metodą otrzymywania powiadomień o działaniach w prywatnych repozytoriach.
