---
question: "Czym jest SCIM w kontekście GitHub?"
documentation: "https://docs.github.com/en/enterprise-cloud@latest/organizations/managing-saml-single-sign-on-for-your-organization/about-scim-for-organizations"
---

- [x] SCIM, czyli System for Cross-domain Identity Management, to protokół zaprojektowany do automatyzacji zarządzania i udostępniania tożsamości. W GitHub SCIM integruje się z zewnętrznymi dostawcami tożsamości (IdPs), aby zarządzać członkostwami w GitHub Enterprise Cloud organizations, używając bazowego adresu URL dla punktów końcowych SCIM do wykonywania operacji, takich jak przeglądanie, zapraszanie i aktualizowanie tożsamości użytkowników.
- [ ] SCIM to system zarządzania użytkownikami wyłącznie dla GitHub, który umożliwia użytkownikom dostosowywanie ich profili i repozytoriów. W GitHub SCIM działa, dostarczając specyficzny dla platformy interfejs do tych dostosowań.
- [ ] SCIM oznacza System for Cross-domain Identity Management. Jest to protokół, który GitHub wykorzystuje do zarządzania wewnętrznymi systemami baz danych, koncentrując się głównie na zabezpieczeniu danych i informacji o użytkownikach bez integracji z jakimikolwiek zewnętrznymi systemami.
- [ ] SCIM to narzędzie specyficzne dla GitHub, używane do poprawy wydajności sieci serwerów GitHub. Działa, zarządzając przepustowością internetu i ruchem w ekosystemie GitHub, zapewniając optymalną wydajność dla wszystkich użytkowników.
