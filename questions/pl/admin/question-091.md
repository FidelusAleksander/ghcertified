---
question: "W jaki sposób można korzystać z zewnętrznych sejfów do zarządzania sekretami w GitHub Actions?"
documentation: "https://developer.hashicorp.com/vault/docs/platform/github-actions"
---

> https://bitwarden.com/help/github-actions-integration/
> https://docs.keeper.io/keeperpam/secrets-manager/integrations/github-actions
- [x] Przechowuj sekrety w zewnętrznym sejfie i uwierzytelniaj z poziomu swojego workflow (za pomocą akcji dostawcy/CLI) używając poświadczeń dostępowych przechowywanych jako sekrety GitHub Actions, aby pobrać je w czasie wykonywania.
- [ ] Bezpośrednia integracja z zewnętrznymi sejfami jest natywnie obsługiwana w GitHub Actions bez potrzeby dodatkowej konfiguracji.
- [ ] Zewnętrzne sejfy powinny być dostępne poprzez osadzanie ich kluczy API w plikach workflow i korzystanie z API lub CLI tego dostawcy.
- [ ] GitHub Actions nie obsługuje korzystania z zewnętrznych sejfów do zarządzania sekretami; wszystkie sekrety muszą być przechowywane w GitHub.
