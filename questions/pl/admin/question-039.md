---
question: "Jak administrator organizacji może ustawić domyślne uprawnienia dla nowych członków w organizacji GitHub?"
documentation: "https://docs.github.com/en/organizations/managing-user-access-to-your-organizations-repositories/managing-repository-roles/setting-base-permissions-for-an-organization#setting-base-permissions"
---

- [x] W sekcji `Access` ustawień organizacji wybierz `Member privileges`, następnie w `Base permissions` wybierz poziom uprawnień i potwierdź zmianę.
- [ ] Domyślne uprawnienia są automatycznie określane przez GitHub na podstawie aktywności organizacji i nie można ich ustawić ręcznie.
- [ ] Poprzez indywidualne ustawienie uprawnień dla każdego nowego członka w momencie dodawania do organizacji, bez dostępnej domyślnej opcji.
- [ ] Domyślne uprawnienia dla nowych członków są ustawiane za pomocą wywołania GitHub API, które określa żądany poziom uprawnień bazowych dla organizacji.
