---
question: "Jakie są kluczowe implikacje włączenia jednokrotnego logowania (SSO) SAML dla organizacji w GitHub Enterprise Cloud?"
documentation: "https://docs.github.com/en/enterprise-cloud@latest/authentication/authenticating-with-single-sign-on/about-authentication-with-single-sign-on"
---

- [x] SAML SSO umożliwia właścicielom organizacji kontrolowanie i zabezpieczanie dostępu do zasobów organizacji.
- [ ] Włączenie SAML SSO automatycznie tworzy nowe konta GitHub dla wszystkich użytkowników IdP.
- [ ] Po włączeniu SAML SSO publiczne repozytoria stają się niedostępne.
- [ ] Włączenie SAML SSO wymaga, aby członkowie uwierzytelniali się za pośrednictwem GitHub, a nie IdP.
