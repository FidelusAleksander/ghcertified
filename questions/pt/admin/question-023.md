---
question: "Qual é o propósito do SCIM e da sincronização de equipes no GitHub?"
documentation: "https://docs.github.com/en/enterprise-cloud@latest/organizations/managing-saml-single-sign-on-for-your-organization/managing-team-synchronization-for-your-organization"
---

- [x] A sincronização de equipes conecta equipes do GitHub com grupos do IdP para gerenciamento de membros, baseando-se no SAML single sign-on, enquanto o SCIM automatiza o provisionamento e gerenciamento de identidades entre sistemas, incluindo a criação e atualização de identidades de usuários e acessos.
- [ ] A sincronização de equipes é um processo automatizado que permite aos usuários gerenciar seus perfis públicos e repositórios através de uma interface específica do GitHub.
- [ ] A sincronização de equipes é usada para segurança interna de dados e gerenciamento de informações do usuário, enquanto o SCIM é uma interface para personalização de perfis de usuários e repositórios.
- [ ] Tanto a sincronização de equipes quanto o SCIM no GitHub são usados para serviços de provisionamento de usuários, convidando não-membros a ingressar em organizações e gerenciar perfis públicos e repositórios.
