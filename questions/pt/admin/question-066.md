---
question: "Por que os webhooks podem ser considerados uma alternativa eficiente ao audit log ou à API polling em determinados casos de uso dentro de organizações do GitHub?"
documentation: "https://docs.github.com/en/organizations/keeping-your-organization-secure/managing-security-settings-for-your-organization/reviewing-the-audit-log-for-your-organization"
---

- [x] Os webhooks fornecem notificações em tempo real para o seu servidor quando eventos específicos ocorrem, o que pode ser mais eficiente do que buscar no audit log ou fazer polling na API para esses eventos.
- [ ] Os webhooks permitem que os administradores da organização revisem e aprovem manualmente cada evento antes que ele seja registrado.
- [ ] Eles fornecem um registro de eventos mais detalhado e abrangente do que o audit log.
- [ ] Os webhooks são o único método para receber notificações sobre ações em repositórios privados.
