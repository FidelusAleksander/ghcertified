---
question: "Como os GitHub Apps podem reagir a eventos específicos e quais são alguns exemplos desses eventos?"
documentation: "https://docs.github.com/en/apps/creating-github-apps/registering-a-github-app/using-webhooks-with-github-apps#subscribing-to-webhook-events"
---

- [x] Os GitHub Apps se inscrevem em eventos por meio de webhooks, que notificam o aplicativo sobre ações específicas, como abertura de pull requests ou criação de issues.
- [ ] Os GitHub Apps utilizam polling para verificar periodicamente novos eventos no GitHub.
- [ ] Os GitHub Apps recebem todos os eventos por padrão e devem filtrá-los no código do aplicativo.
- [ ] Os GitHub Apps usam uma conexão WebSocket para receber notificações em tempo real de todos os eventos do repositório.
