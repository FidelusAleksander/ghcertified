---
question: "Quais dessas ações garantirão que, antes de qualquer envio para a branch `main`, as alterações sejam aprovadas por, pelo menos, duas pessoas?"
documentation: "https://docs.github.com/en/repositories/configuring-branches-and-merges-in-your-repository/managing-protected-branches/about-protected-branches#require-pull-request-reviews-before-merging"
---

- [x] Criar uma regra de proteção de branch para a branch `main`
- [x] Na regra de proteção de branch, exigir um pull request antes de mesclar
- [x] Na regra de proteção de branch, exigir pelo menos 2 aprovações antes de mesclar um pull request
- [ ] Criar uma regra de proteção de implantação para a branch `main`
- [ ] Na regra de proteção de implantação, exigir pelo menos 2 aprovações antes de mesclar um pull request
- [ ] Configurar um arquivo CODEOWNERS no repositório
- [ ] Garantir que as verificações de status estejam passando antes de mesclar um pull request para a branch `main`
