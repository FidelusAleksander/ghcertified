---
question: "O que é o GitHub Flow?"
documentation: "https://docs.github.com/en/get-started/quickstart/github-flow"
---

- [x] Refere-se a um fluxo de trabalho baseado em branch onde os desenvolvedores criam uma branch para cada nova alteração em que estão trabalhando e, em seguida, abrem pull requests para que seu código seja revisado, testado e mesclado na branch principal.
- [ ] É uma prática onde, toda vez que um commit é enviado para a branch principal, uma nova release é automaticamente criada e implantada.
- [ ] É uma prática onde, toda vez que um commit é enviado para a branch principal, um conjunto de pipelines CI/CD é acionado para executar testes e validar as alterações no código.
- [ ] Refere-se a um fluxo de trabalho de uma única branch onde os desenvolvedores trabalham em suas features na branch `master` e depois têm seu código de feature revisado, testado e comprometido.
