---
question: "Como funciona a sincronização entre projetos do GitHub, issues e pull requests?"
documentation: "https://docs.github.com/en/issues/planning-and-tracking-with-projects/learning-about-projects/about-projects#staying-up-to-date"
---

- [x] As atualizações nas issues e pull requests serão refletidas automaticamente nos GitHub Projects. Esta integração funciona nos dois sentidos, de modo que, ao alterar as informações sobre uma pull request ou issue em seu projeto, a pull request ou issue reflete essa informação.
- [ ] As atualizações nas issues e pull requests não serão refletidas automaticamente nos GitHub Projects, devendo ser acionadas manualmente pelo usuário.
- [ ] A sincronização das informações funciona apenas em uma direção - do projeto para as issues e pull requests. As atualizações feitas nas issues e pull requests não serão espelhadas automaticamente nos itens do GitHub Project.
- [ ] A sincronização das informações funciona apenas em uma direção - das issues e pull requests para os projetos. As atualizações feitas nos itens do projeto não serão espelhadas automaticamente nas issues e pull requests.
