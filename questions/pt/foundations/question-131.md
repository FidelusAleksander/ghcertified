---
question: "Qual afirmação descreve melhor a diferença entre clonar um repositório e fazer fork de um repositório?"  
draft: false  
documentation: "https://docs.github.com/pt/pull-requests/collaborating-with-pull-requests/working-with-forks/about-forks"
---

- [ ] Quando você clona um repositório, as alterações que você envia vão diretamente para o repositório original. Fazer fork exige uma cópia local separada.  
> Clonar e fazer fork resultam em cópias locais separadas, mas enviar alterações para um clone exige a permissão correta no repositório original.  
- [x] Fazer fork cria uma cópia no seu GitHub, enquanto clonar cria uma cópia local no seu computador sem vincular a propriedade à sua conta.  
> **Correto**. Um fork é uma cópia no nível do GitHub que aparece na sua conta, enquanto um clone é apenas uma cópia local.  
- [ ] Clonar sempre transfere a propriedade do repositório, enquanto fazer fork não.  
- [ ] Somente repositórios privados podem ser objeto de fork, enquanto apenas repositórios públicos podem ser clonados.
