---
question: "Vários repositórios públicos aos quais você está contribuindo não têm a opção de proteção contra push de segredos habilitada. O que você pode fazer para se proteger de acidentalmente enviar segredos para esses repositórios?"
documentation: "https://docs.github.com/en/code-security/secret-scanning/push-protection-for-users#about-push-protection-for-users"
---

- [x] Habilitar `Push protection for yourself` nas configurações da sua conta pessoal no GitHub
- [ ] Baixar o plugin web de proteção contra push do GitHub
- [ ] Não é possível, a proteção contra push deve estar habilitada no nível do repositório, organização ou enterprise
- [ ] Adicionar os arquivos contendo segredos ao arquivo `.gitignore` em todos os repositórios
