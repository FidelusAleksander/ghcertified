---
question: "Como você pode evitar que commits contendo credenciais de provedores de nuvem sejam enviados para o GitHub?"
documentation: "https://docs.github.com/en/code-security/concepts/secret-security/push-protection"
---

- [x] Ativar uma regra de proteção contra push de análise de segredos para seu repositório ou organização.
- [ ] Incluir um arquivo `.gitignore` no seu repositório que ignore arquivos contendo segredos.
- [ ] Criar um GitHub Action que analise seus commits em busca de segredos antes de serem enviados para o GitHub.
- [ ] Ativar uma regra de proteção de branch para seu repositório.
